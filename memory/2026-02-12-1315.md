## Cron Model Clarification

Zsolt asked what model runs the daily email summary.

**Answer**: The `daily-email-summary` cron has:
- `sessionTarget: "main"` 
- `payload.kind: "systemEvent"`

This means it injects the prompt into the **main session**, so whatever model is running there handles it. Today that was me (Opus 4.5). It's NOT an isolated session with a different model.
