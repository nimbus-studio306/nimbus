## Agent Architecture Deep Dive (Evening Session ~20:00-23:00 UTC)

### Four Architectures Identified

**1. Nimbus-VM (OpenClaw Native)**
- Uses `sessions_spawn` and `sessions_send`
- 9 subagents as OpenClaw agents
- Simple, built-in, no external dependencies
- Missing: HTTP API, inter-agent messaging

**2. Nimbus Studio (Mac Studio Bus System)**
- Filesystem-based bus (inbox/outbox)
- HTTP API on port 18785 (`nimbus serve`)
- Python library: `nimbus.bus.lib`
- Agents use `claude-code` or `kimi` CLI runtime
- **Proven to work**: Built nimbus.kallossoft.co website from voice

**3. Mac Studio AgentMaster**
- `dispatch.sh` calls `claude-msg` / `kimi-msg`
- Direct CLI spawning
- Subagents have CLAUDE.md instruction files
- Cost tracking via dispatch-costs.jsonl

**4. Nimbus-VM AgentMaster (Adapted)**
- dispatch.sh uses `sessions_spawn`
- 9 subagents created as OpenClaw agents
- Needs: CLAUDE.md files for each subagent
- Needs: Inter-agent communication layer

### Key Discovery: Nimbus Package Architecture

**Standalone Python Service** (`~/.openclaw/services/nimbus/`):
- FastAPI HTTP server (`nimbus serve` on port 18785)
- Independent from OpenClaw but uses same database
- CLI: `nimbus agents`, `nimbus bus`, `nimbus dispatch`, etc.
- OpenClaw plugin is thin wrapper calling HTTP API

**Database: SHARED PostgreSQL**
- Same host: 81.0.107.97:5433
- Mac Studio uses: `project_openclaw_studiokallos`
- Nimbus-VM uses: `project_openclaw_nimbus`
- Tables: Mix of shared (nodes, edges, sessions) + new (tool_inventory, notes)

**Tables Created by Nimbus Service:**
- `tool_inventory` — tracks CLI tools, packages, MCPs
- `tool_version_history` — version change log
- `notes`, `documents`, `memories` — content storage
