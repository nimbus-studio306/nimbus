# 2026-02-11 — Voice Message Pattern Discovery

## Voice Message Sending (CONFIRMED WORKING)

### The Problem
- `tts()` tool generates audio files at `/tmp/tts-<random>/voice-<timestamp>.mp3`
- Output is `MEDIA:/path/to/file.mp3` — this path string doesn't auto-send
- Need to explicitly send the audio file via WhatsApp

### The Solution
1. **Generate TTS:** `tts({ text: "..." })` → returns `MEDIA:/tmp/tts-xxx/voice-xxx.mp3`
2. **Base64 encode:** `base64 -w0 /tmp/tts-xxx/voice-xxx.mp3`
3. **Send via message tool:**
```javascript
message({
  action: "send",
  channel: "whatsapp", 
  target: "+36703407845",
  message: "🎤",  // placeholder, required
  buffer: "data:audio/mp3;base64,<BASE64_AUDIO>",
  contentType: "audio/mp3",
  asVoice: true
})
```

### Key Details
- `buffer` must include the data URL prefix: `data:audio/mp3;base64,...`
- `asVoice: true` makes it play as a voice note (not file attachment)
- `message` param is required (use emoji placeholder)
- Works consistently — tested multiple times

## Session Recovery Fix (04:30 UTC)

### The Problem
Main session was stuck — incomplete tool call cycle:
1. User message (`/compact`)
2. Assistant message with `stopReason: "toolUse"` (exec to base64 encode audio)
3. Tool result returned (huge base64 string)
4. **Missing:** Assistant completion to process tool result

Session appeared frozen — new messages caused timeouts.

### The Fix
```bash
# Backup
cp session.jsonl session.jsonl.bak

# Remove incomplete turn (last 2 lines: toolUse assistant + toolResult)
total=$(wc -l < session.jsonl)
keep=$((total - 2))
head -n $keep session.jsonl.bak > session.jsonl
```

### Result
- Session reduced from 312 → 310 lines
- Ends cleanly on user message
- Model responds normally to new messages
- Verified with test message: "Yep — I'm here and I can hear you clearly."

### Lesson Learned
When a session is stuck on incomplete tool calls:
1. Check last few lines with `jq` for `stopReason: "toolUse"` followed by `toolResult` but no completion
2. Remove the incomplete cycle (both messages)
3. Session resumes from clean user message state
