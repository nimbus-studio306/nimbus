## `Deeper` vs `deeper` Agent Folder Investigation
- Investigated the presence of two agent folders: `Deeper` (capitalized) and `deeper` (lowercase).
- **Initial incorrect diagnosis:** Assumed one was a mistake and proposed a migration.
- **Final correct diagnosis:** This is intentional OpenClaw behavior.
  - The `openclaw.json` config defines the agent `id: Deeper`, which correctly points to `Deeper/workspace/` for identity/memory files.
  - At runtime, OpenClaw normalizes the agent ID to lowercase for session management, storing live session data in `deeper/sessions/`.
- **Conclusion:** Both folders are actively used by the same agent for different purposes. No cleanup or migration is needed. This is a key insight into the system's architecture.
