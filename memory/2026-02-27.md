## Key Architectural Insights

### Agent Master (Mac Studio) vs Original Nimbus Design
- **Agent Master:** Ephemeral subagents (spawn → run → die)
- **Original Nimbus:** Persistent specialist sessions
- **Communication:** Agent Master uses stdout, Nimbus design used file-based
- **Both have value:** Ephemeral for quick tasks, persistent for deep work

### Working Solution for Nimbus App
The Nimbus app needed to be in the OpenClaw gateway's built-in static file directory, not served separately. The gateway serves its own Control UI from `dist/control-ui/`, so placing the app there makes it accessible via the same tunnel.
