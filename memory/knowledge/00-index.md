# Knowledge Base Index
> Created: 2026-02-03
> Purpose: Structured knowledge that's searchable via memory_search
> Structure: taxonomy-inspired (vocabularies → categories → content)

## Why Here?
Memory search only indexes MEMORY.md + memory/*.md (recursive).
Files in reference/ or other dirs are invisible to vector search.
Keeping knowledge here = it's always findable.

## Organization

### Self-Knowledge (how I work)
- `self-environment.md` — my tools, APIs, capabilities, paths
- `self-memory-system.md` — how my memory/search works (from source)
- `self-system-prompt.md` — how my behavior gets assembled
- `openclaw-architecture.md` — source code structure map (building)

### Meta
- This index — overview of what exists and where
