## Knowledge Graph Context Injection (00:30-01:10 UTC → Feb 13)

### Problem Identified
Auto-injecting full graph context on every message caused:
1. Context pollution with generic entities (WhatsApp, Telegram, OpenClaw)
2. Irrelevant past messages (cron jobs, heartbeats)
3. Slowdown from DB queries on every message
4. Loss of focus on actual conversation topic

### Solution Implemented: Hint-Only Mode
Instead of auto-querying the database, the hook now just injects a simple question:
`[Knowledge Graph Hint: Consider if this conversation would benefit from historical context...]`

Then I decide whether to call `knowledge_graph` tool myself.

**Flow:**
1. Message comes in
2. Hook injects lightweight hint (no DB query)
3. I think: "Does this need historical context?"
4. If YES → I call knowledge_graph tool → get context → respond with knowledge
5. If NO → respond directly

### Files Changed
- `plugins/knowledge-graph/src/context-injection-hook.ts` - hint-only mode
- `update-graph-context.py` - now includes message content with entities
