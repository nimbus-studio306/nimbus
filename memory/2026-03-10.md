**Status:** ✅ DONE — Comprehensive research completed

**Details:**
- Created `memory/knowledge/research-whatsapp-call-transcription.md` (9.6KB)
- Documented key challenges (raw digital audio vs mic recording)
- Researched 3 solution options:

**Option A: Browser-Based (Web Audio API)**
- Uses `getDisplayMedia()` to capture system audio
- Works for WhatsApp Web only
- No system-level audio routing needed
- Estimated: 2-3 days implementation

**Option B: Desktop Audio Capture (Electron)**
- System-level capture via WASAPI (Windows), Core Audio (macOS), PipeWire (Linux)
- Works for WhatsApp Desktop
- Higher quality, lower latency
- Estimated: 1-2 weeks implementation

**Option C: Mobile Methods**
- Screen recording apps (Aloop, etc.)
- Android: ADB command line
- iOS: Built-in screen recording
- Pros: Works on mobile, no desktop needed
- Cons: User visibility, quality issues

**Recommended Architecture:**
- **Phase 1:** Browser-based POC for WhatsApp Web (quick win)
- **Phase 2:** Desktop production solution (cross-platform)

**Integration with Gemini STT:**
- Streaming transcription via WebSocket or chunked HTTP
- Audio chunk size: 1-2 seconds
- Format: WAV + PCM
- Language auto-detection (Hungarian/English)
