## What Happened
Late night session building real-time voice call into nimbus-app. Zsolt was hands-on testing throughout.

## Voice Call Architecture (Final State)
- **Gateway plugin**: `plugins/voice-proxy/` — registers `/voice/stt` and `/voice/tts` HTTP routes on gateway port 18789
- **STT**: Gemini 2.5 Flash (same as WhatsApp) — browser records audio via MediaRecorder + VAD, sends blob to `/voice/stt`, gets text back
- **TTS**: Edge TTS `en-US-MichelleNeural` (same as WhatsApp) — text sent to `/voice/tts`, MP3 audio returned
- **Voice in constructor**: EdgeTTS voice MUST be passed in constructor `new EdgeTTS({voice, lang, outputFormat})`, NOT in `ttsPromise()` options — different API!
- **Session**: Uses dedicated `voice-call` session key (isolated from main)
- **Plugin config**: Added to `plugins.load.paths` and `plugins.entries.voice-proxy.enabled: true`

## Key Technical Decisions
- **No browser speech APIs**: Zsolt explicitly rejected SpeechRecognition and SpeechSynthesis — wants server-side only
- **No API keys in browser**: Rejected `gemini_key=` URL param approach — the plugin handles everything server-side
- **Gateway plugin approach**: Plugin registers HTTP routes on same port (18789), no extra ports, no Cloudflare changes needed
- **chat.send only accepts image attachments**: Audio attachments get DROPPED by `parseMessageWithAttachments()` — can't send audio through WebSocket
- **Streaming TTS**: Text-to-speech starts as first sentence arrives from agent, not waiting for full response. Sentence boundary detection + fallback to comma breaks at 100 chars + force break at 200 chars

## VAD Settings (Current)
- SILENCE_THRESHOLD: 8 (RMS level)
- SPEECH_CONFIRM_MS: 200ms sustained speech before counting
- SILENCE_DURATION: 3500ms (3.5s solid silence before sending)
- MIN_RECORDING_MS: 2000ms minimum recording
- MIN_AUDIO_BYTES: 2000
