### Gmail Hook Duplicate Issue Identified
The Gmail hook fired 7-8 times for the same single email because:
1. Gmail Pub/Sub triggers on multiple events per email (created, sent, label changes)
2. Each event gets a different historyId/messageId
3. The hook sessionKey uses `{{messages[0].id}}` but these IDs differ per notification
4. Pub/Sub has "at-least-once delivery" which can duplicate

This caused a "traffic jam" - multiple agent sessions spawning simultaneously, fighting for the session lock, causing timeout errors.

Fix options discussed:
- Add deduplication by subject+recipient+timestamp
- Filter to only specific label events
- Track seen thread IDs
